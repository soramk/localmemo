# Issue: ローカル画像のプレビュー表示対応

## ステータス
**Closed (完了)**

## 概要
エディタ内に画像をドラッグ＆ドロップして `.images` フォルダに保存されたローカル画像が、Markdownプレビュー画面で正しく表示されない（リンク切れになる）不具合を修正しました。

## 原因と対応
ローカルPC内のファイルシステム（File System Access API経由で取得したもの）は、ブラウザが通常のHTTPリクエスト（`<img src="./.images/...">` など）で読み込むことができません。そのため、ドラッグ＆ドロップで保存した画像のパスが正しくMarkdownに挿入されていても、ブラウザのセキュリティ上の制約でプレビューには表示されない状態でした。

これを解決するため、以下のような処理を実装しました。
1. **カスタム画像レンダラーの実装**: `react-markdown` で `img` タグを描画する際に、通常の描画をフックして専用の `CustomImage` コンポーネントを使用するように変更しました。
2. **非同期ファイルロードとObjectURLへの変換**: Markdown内の画像パスを検知した際、現在開いているファイルのディレクトリハンドルから対象の画像ファイル（`.images` 内のファイル）を直接検索し、読み込みます。
3. **安全な表示**: 読み込んだ画像データをブラウザ内の一時的なURL（`Blob URL` / `Object URL`）に変換し、それを `img` タグの `src` に渡すことで、完全なローカル環境でも安全かつ高速に画像を表示できるようになりました。

## 完了状態
上記処理により、ドラッグ＆ドロップしたローカル画像がプレビュー画面で瞬時に表示されることを確認しました。ビルドエラーもなく正常に動作するため本Issueをクローズします。
