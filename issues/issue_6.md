# Issue: GitHub Pages上の画面が真っ白になる原因（リポジトリ設定の未変更）

## ステータス
**Closed (完了)**

## 概要
提供いただいたURL (`https://soramk.github.io/localmemo/`) のソースコードを確認したところ、画面が真っ白になる原因が判明しました。

## 原因
現在、GitHub Pagesが**「ビルド前の生のソースコード（未変換のプログラム）」**をそのまま配信してしまっています。
WebブラウザはViteやReactのプログラム（JSX形式）を直接実行できないため、描画処理が走らずに画面が真っ白になっています。

作成した自動デプロイ用ワークフロー (`.github/workflows/deploy.yml`) は、ブラウザが理解できる形式に「ビルド（変換）」してから公開するためのものですが、現在のGitHubリポジトリの設定では、そのワークフローの結果が使われていません。

## 解決手順（ユーザー側での操作）
この問題を解決するには、GitHubのリポジトリ設定画面で以下の操作を行ってください。

1. GitHubの `soramk/localmemo` リポジトリ画面を開きます。
2. 上部のタブから **Settings（設定）** をクリックします。
3. 左側のメニューから **Pages** をクリックします。
4. **Build and deployment** の項目にある **Source** のプルダウンメニューを確認します。
   - おそらく現在 `Deploy from a branch` になっていると思われます。
5. このプルダウンメニューをクリックし、**`GitHub Actions`** に変更してください。

変更後、先ほど作成したワークフローが自動的に走り出し、数分後にビルド済みの正しいファイルが公開されます。完了後に再度URLを開いていただくと、画面が表示されます。

## 完了状態
原因の特定と解決策の提示が完了したため、本Issueをクローズします。上記の手順を実施してください。
