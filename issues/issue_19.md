# Issue: 汎用ファイル（GitHub/コード/メディア）のサポート拡充

## ステータス
**Closed (完了)**

## 概要
GitHubでの開発や日常的なメモ管理で利用される多様なファイル拡張子への対応を大幅に強化しました。また、非テキストファイル（画像、PDF、動画、音声）を直接閲覧できるマルチビューワー機能を実装しました。

## 実施内容

1. **対応拡張子の拡充**
   - ファイルツリーに表示されるファイルを大幅に増やしました。
   - **コード/設定**: `.js`, `.ts`, `.jsx`, `.tsx`, `.yml`, `.yaml`, `.json`, `.html`, `.css`, `.py`, `.sh`, `.env`, `.gitignore` など
   - **ドキュメント**: `.pdf`, `.csv`, `.log` など
   - **メディア**: `.png`, `.jpg`, `.gif`, `.webp`, `.svg`, `.mp4`, `.mp3` など

2. **マルチビューワー機能の実装**
   - ファイルを選択した際、拡張子に応じて自動的に最適なビューワーに切り替わります。
   - **テキスト系**: 従来通りエディタで開き、編集可能です。
   - **画像**: 中央にフィットして表示されます。
   - **PDF**: ブラウザ標準のPDFビューワー（iframe）により、全画面で閲覧可能です。
   - **動画/音声**: 再生コントローラーが表示され、アプリ内で直接視聴できます。

3. **パフォーマンス最適化**
   - ディレクトリ読み込み時、`node_modules` や `.git` などの膨大なファイルを含むフォルダを自動的にスキップするようにし、プロジェクトフォルダを開いた際の動作を高速化しました。

## 完了状態
主要な全拡張子の表示と、各メディア（画像/PDF/動画）の正常なプレビュー動作を確認したため、本Issueをクローズします。
